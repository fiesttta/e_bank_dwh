# 🏦 Мой проект: Банковское хранилище (DWH) и ETL

Это мой учебный проект, в котором я с нуля построил архитектуру данных для небольшого е_банка. Здесь всё — от генерации случайных транзакций до умных триггеров и обработки Big Data через PySpark.

## 🛠 Что внутри? (Стек)
* **База:** PostgreSQL (тут живет наша база и данные).
* **Языки:** Python (для генерации данных и ETL) + SQL (от простых запросов до сложных оконных функций).
* **Big Data:** PySpark (потренировался обрабатывать данные в стиле больших корпораций).
* **Инструменты:** pgAdmin 4, VS Code и Git.

## 📁 Как всё устроено
* `data_generator.py` — Создает тысячи реалистичных транзакций, клиентов и счета.
* `etl_pipeline.py` — Берет сырые данные, подтягивает курсы валют и пересчитывает всё в рубли для витрины.
* `etl_pyspark_colab_example` — полигон для PySpark. Тот же ETL, но на мощностях Big Data (запускал через Google Colab).
* `migrations/` — Триггер `7_balance_trigger.sql`, который сам обновляет баланс счета, как только падает новая транзакция.
* `analytics/` — папка с моими SQL-заготовками: оконные функции, CTE, представления (Views) и т.д.

## 🚀 Что было изучено
1. **Проектирование схемы:** Разделил базу на "быструю" (OLTP) для транзакций и "аналитическую" (OLAP) для отчетов.
2. **Написание умного SQL:** Оконные функции для поиска первых транзакций, сложные джоины и CTE.
3. **Автоматизирование базу:** Написал триггер на PL/pgSQL — теперь база сама считает остатки на счетах без моей помощи.
4. **Работа с Big Data:** Разобрался с ленивыми вычислениями в Spark и тем, как объединять огромные таблицы.

## ⚙️ Как запустить у себя
1. Клонируй репо.
2. Создай базу `e_bank` в PostgreSQL.
3. Запусти `python data_generator.py` — он наполнит базу.
4. Запусти `python etl_pipeline.py` — и у тебя появится готовая витрина в рублях.
